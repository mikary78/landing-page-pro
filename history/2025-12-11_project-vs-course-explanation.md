## 프로젝트와 코스의 차이점

### 프로젝트 (Project)
**용도**: 단일 교육 자료를 빠르게 생성

**특징**:
- 하나의 주제에 대한 완전한 교육 자료
- 예: "파이썬 기초 수업안", "마케팅 전략 슬라이드"
- 6단계 AI 콘텐츠 생성 (커리큘럼 설계 → 수업안 작성 → 슬라이드 구성 → 실습 템플릿 → 평가/퀴즈 → 최종 검토)
- 빠른 생성 및 사용

**사용 시나리오**:
- 단일 수업 자료가 필요한 경우
- 빠르게 콘텐츠를 생성하고 싶은 경우
- 특정 주제에 대한 완성된 자료가 필요한 경우

### 코스 (Course)
**용도**: 여러 레슨으로 구성된 체계적인 교육 과정

**특징**:
- 여러 모듈과 레슨으로 구성
- 예: "파이썬 입문 4주 과정" (12개 레슨으로 구성)
- 각 레슨은 프로젝트를 참조하여 콘텐츠 생성
- 체계적인 커리큘럼 관리

**구조**:
```
코스 (Course)
├── 모듈 1 (Module)
│   ├── 레슨 1 (Lesson) → 프로젝트 참조
│   ├── 레슨 2 (Lesson) → 프로젝트 참조
│   └── 레슨 3 (Lesson) → 프로젝트 참조
├── 모듈 2 (Module)
│   ├── 레슨 4 (Lesson) → 프로젝트 참조
│   └── 레슨 5 (Lesson) → 프로젝트 참조
└── ...
```

**사용 시나리오**:
- 여러 레슨으로 구성된 교육 과정이 필요한 경우
- 체계적인 커리큘럼을 관리하고 싶은 경우
- 단계별 학습 경로를 제공하고 싶은 경우

## 프로젝트 생성이 필요한 이유

### 1. 단독 사용
- 코스 없이도 단일 프로젝트로 빠르게 콘텐츠 생성 가능
- 간단한 교육 자료가 필요한 경우 프로젝트만으로 충분

### 2. 코스의 레슨 콘텐츠 생성
- 코스의 각 레슨은 프로젝트를 참조하여 콘텐츠 생성
- 레슨 생성 시 자동으로 프로젝트가 생성되고 AI 콘텐츠가 생성됨

### 3. 유연성
- 프로젝트는 코스에 포함되거나 독립적으로 사용 가능
- 사용자의 필요에 따라 선택적으로 사용

## 사용 가이드

### 프로젝트 생성 사용
1. 대시보드 → "내 프로젝트" 탭
2. "새 프로젝트 생성" 클릭
3. 프로젝트 정보 입력 및 AI 콘텐츠 생성

### 코스 생성 사용
1. 대시보드 → "내 코스" 탭
2. "새 코스 생성" 클릭
3. 코스 정보 입력
4. 코스 빌더에서 모듈 및 레슨 추가
5. 각 레슨에 대해 "이 레슨을 AI로 생성하기" 클릭 (자동으로 프로젝트 생성)
