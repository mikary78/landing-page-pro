# 개발 히스토리

이 폴더는 **중대한 변화**가 있을 때만 문서화합니다.

## 중대한 변화 기준
1. 신기능 구현
2. DB 구조 수정
3. UI 레이아웃 대규모 변경

## 문서 형식

각 문서는 다음 형식을 따릅니다:

```markdown
# [날짜] - [변경 제목]

## 사용자 요구사항
- 사용자가 요청한 내용

## 구현 답변
- 개발자가 답한 내용

## 수정 내역 요약
- 실제 수정된 파일 및 내용
- 추가된 기능
- 변경된 로직

## 테스트
- 구현한 기능의 테스트 코드 위치
- 테스트 시나리오
```

## 일반 변경사항

중대한 변화가 아닌 경우, Git 커밋 메시지에 충실하게 작성합니다.
