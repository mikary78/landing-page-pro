## 1. 개요

### 1.1 문서 목적

이 문서는 AI AutoPilot Landing Page Pro 프로젝트의 End-to-End(E2E) 테스트 케이스를 상세하게 정의합니다. 프론트엔드와 백엔드의 통합 테스트를 통해 사용자 관점에서의 기능 검증을 목표로 합니다.

### 1.2 테스트 범위

| 범위 | 설명 |
|------|------|
| **UI 테스트** | 페이지 로드, 요소 표시, 사용자 인터랙션 |
| **API 테스트** | Azure Functions 엔드포인트 통합 |
| **통합 테스트** | 사용자 플로우, 인증, 데이터 흐름 |

### 1.3 테스트 도구

- **Selenium WebDriver**: 기본 E2E 테스트
- **Playwright**: Azure Functions 통합 테스트
- **Vitest**: 테스트 러너

## 2. 테스트 환경

### 2.1 환경 변수 설정

```env
# E2E 테스트 기본 URL
E2E_BASE_URL=http://localhost:5173

# 테스트 계정 정보 (민감 정보 주의)
E2E_TEST_EMAIL=test@example.com
E2E_TEST_PASSWORD=testpassword123

# 헤드리스 모드 (CI 환경)
HEADLESS=true
CI=true

# Azure Functions URL
VITE_AZURE_FUNCTIONS_URL=https://func-landing-page-pro.azurewebsites.net
```

### 2.2 사전 조건

1. 개발 서버 실행 중 (`npm run dev`)
2. Azure Functions 배포 완료
3. 유효한 테스트 계정 준비
4. 브라우저 드라이버 설치

## 3. 테스트 케이스 분류

### 3.1 우선순위 정의

| 우선순위 | 설명 | 실행 빈도 |
|----------|------|-----------|
| **P0 (Critical)** | 핵심 기능, 서비스 중단 시 사용 불가 | 매 배포 |
| **P1 (High)** | 중요 기능, 사용자 경험에 큰 영향 | 매 배포 |
| **P2 (Medium)** | 일반 기능, 대체 경로 존재 | 일일 |
| **P3 (Low)** | 부가 기능, 미미한 영향 | 주간 |

### 3.2 테스트 유형

| 유형 | 설명 |
|------|------|
| **Smoke Test** | 기본 기능 빠른 검증 |
| **Regression Test** | 기존 기능 영향 검증 |
| **Sanity Test** | 특정 기능 집중 검증 |

### 6.2 SC-001: 신규 사용자 회원가입 플로우

**플로우**: 랜딩 페이지 → 회원가입 → 대시보드

**사전 조건**:
- 신규 이메일 주소 준비

**테스트 단계**:

| 단계 | 액션 | 예상 결과 |
|------|------|----------|
| 1 | 랜딩 페이지 접속 | 페이지 정상 로드 |
| 2 | 로그인/회원가입 버튼 클릭 | /auth 페이지로 이동 |
| 3 | 회원가입 모드로 전환 | 회원가입 폼 표시 |
| 4 | 이메일, 비밀번호 입력 | 입력값 반영 |
| 5 | 회원가입 제출 | 처리 중 표시 |
| 6 | 인증 완료 | 대시보드로 리다이렉트 |

| 3 | 제목, 설명 입력 | 입력값 반영 |
| 4 | AI 모델 선택 | 선택값 반영 |
| 5 | 프로젝트 생성 제출 | 로딩 상태 표시 |

## 7. 테스트 실행 방법

### 7.1 Selenium WebDriver 테스트

```bash
# 모든 E2E 테스트 실행
npm run test:e2e

# 특정 테스트 파일 실행
npm run test:e2e -- src/test/e2e/index.test.ts
npm run test:e2e -- src/test/e2e/auth.test.ts
npm run test:e2e -- src/test/e2e/dashboard.test.ts

# UI 모드로 실행 (대화형)
npm run test:e2e:ui
```

### 7.2 Playwright 테스트

```bash
# 기본 실행
npm run test:playwright

# UI 모드 (대화형)
npm run test:playwright:ui

# 헤드 모드 (브라우저 표시)
npm run test:playwright:headed

# 디버그 모드
npm run test:playwright:debug

# 테스트 리포트 보기
npm run test:playwright:report
```

### 7.3 CI/CD 환경 실행

```bash
# 환경 변수 설정
export HEADLESS=true
export CI=true

# 테스트 실행
npm run test:e2e
npm run test:playwright
```
